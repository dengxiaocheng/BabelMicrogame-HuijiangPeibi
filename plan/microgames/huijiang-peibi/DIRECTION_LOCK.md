# Direction Lock: 灰浆配比

## One Sentence
玩家在材料短缺和赶工压力下调配灰浆，决定墙体是稳固、开裂，还是被暂时遮掩。

## Core Loop
1. 执行核心循环：查看今日材料 -> 调配灰浆 -> 搅拌 -> 涂抹墙段 -> 观察表面反馈 -> 抽检或下一墙段
2. 按 20 分钟节奏推进，分四个节奏阶段：
   - 阶段 A（0-5min）：正常配比教学，材料充足，玩家熟悉滑杆与搅拌
   - 阶段 B（5-10min）：材料开始短缺，玩家需要在不理想比例间权衡
   - 阶段 C（10-15min）：外部压力（雨水/干燥/催工）叠加，抽检风险上升
   - 阶段 D（15-20min）：收束阶段，根据累积 wall_quality 和 inspection_risk 进入结算

## Must Keep
- 核心机制必须保持：比例调配 + 搅拌时间 + 隐藏质量
- 核心循环必须保持：查看今日材料 -> 调配灰浆 -> 搅拌 -> 涂抹墙段 -> 观察表面反馈 -> 抽检或下一墙段
- 20 分钟结构只作为节奏，不扩成大项目
- 玩家必须通过操作场景对象（滑杆、搅拌动作）完成核心交互，不是点按钮

## Must Not Add
- 不做完整建筑模拟；核心是配比误差如何变成质量风险
- 不新增第二套主循环
- 不把小游戏扩成长期经营或开放世界
- 不加 NPC 对话系统或分支叙事树

## Required State
- `water`：当前水量，0-100
- `sand`：当前砂量，0-100
- `lime`：当前灰量，0-100
- `wall_quality`：已涂墙段的累积质量，0-100
- `inspection_risk`：抽检风险，0-100

## Success
在限定时长内完成主循环，并稳定进入至少一个可结算结局：
- 结局 S：wall_quality >= 70 且 inspection_risk < 40 → 墙体通过验收
- 结局 A：wall_quality >= 50 且 inspection_risk < 60 → 勉强过关
- 结局 B：wall_quality < 50 或 inspection_risk >= 60 → 返工或被发现

## Failure
关键状态崩溃（water/sand/lime 三个同时归零），或在本轮主循环中被系统淘汰
