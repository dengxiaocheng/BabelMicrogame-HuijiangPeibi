# MINI GDD: 灰浆配比

## Scope

- runtime: web（浏览器单页，无需后端）
- duration: 20min
- project_line: 灰浆配比
- single_core_loop: 查看今日材料 -> 调配灰浆 -> 搅拌 -> 涂抹墙段 -> 观察表面反馈 -> 抽检或下一墙段
- player_role: 灰浆工，受命在材料受限条件下完成墙段涂抹

## Core Loop
1. 执行核心循环：查看今日材料 -> 调配灰浆 -> 搅拌 -> 涂抹墙段 -> 观察表面反馈 -> 抽检或下一墙段
2. 每轮循环对应一面墙段；20 分钟内完成多面墙段
3. 节奏阶段：
   - 阶段 A（教学期）：材料充足，目标稠度范围宽，反馈宽容
   - 阶段 B（短缺期）：至少一种材料减少，配比约束收紧
   - 阶段 C（压力期）：天气/催工事件叠加，抽检概率上升
   - 阶段 D（收束期）：最终墙段，进入结算判定

## State

- `water`：0-100，每次调配消耗
- `sand`：0-100，每次调配消耗
- `lime`：0-100，每次调配消耗
- `wall_quality`：0-100，每面墙段质量累积
- `inspection_risk`：0-100，质量差或遮掩行为推高风险

## End Conditions

| 条件 | 结局 |
|------|------|
| wall_quality >= 70 且 inspection_risk < 40 | S：墙体通过验收 |
| wall_quality >= 50 且 inspection_risk < 60 | A：勉强过关 |
| wall_quality < 50 或 inspection_risk >= 60 | B：返工或暴露 |
| water/sand/lime 同时为 0 | 系统淘汰 |

## UI

- 单屏布局：左侧场景（搅拌槽+墙段），右侧状态面板（材料余量+稠度条+风险指示）
- 操作区：三个比例滑杆（水/砂/灰）+ 按住搅拌按钮 + 涂抹按钮
- 反馈区：墙面表面即时渲染变化、稠度仪表、抽检风险条
- 不加多余菜单和后台页

## Content

- 事件池（每个事件只推一个核心循环压力）：
  - 材料类：砂子含泥量高、石灰受潮、水源被截
  - 天气类：突降阵雨（需加速）、暴晒（干得太快）
  - 人为类：催工命令、抽检通知、同伴帮忙/干扰
- 一次只验证一条 Babel 创意线：比例误差 → 质量风险 → 遮掩或返工

## Constraints

- 总体规模目标控制在 5000 行以内
- 单个 worker 任务必须服从 packet budget
- 如需扩线，交回 manager 重新拆
